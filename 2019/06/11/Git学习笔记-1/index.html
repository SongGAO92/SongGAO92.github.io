0. 历史

VCS version control system

本地式版本控制系统 local vcs
00-01
集中式 centralized vcs
00-02
优点：

  1. 每个人都可以在一定程度上看到项目中其他人正在做什么
  2. 管理员可以轻松掌握每个开发者权限
  3. 管理起来比在各个客户端维护本地数据库来的容易
    缺点：
  4. 如果中央服务器故障，就都无法进行更新了
  5. 磁盘故障，且没有做备份的话，会有丢失数据的风险

分布式 Distributed vcs
00-03
客户端并不只是提取最新版本的文件快照，而是把代码仓库完整的镜像下来。这样一来，任何一处协同工作用的服务器发生故障，事后都可以用任何一个镜像出来的本地仓库恢复

Git特点：
优秀的存储能力
性能好
开源
容易

git github 和 gitlab

1. 安装

官网安装过程如下：
Debian/Ubuntu

For the latest stable version for your release of Debian/Ubuntu

apt-get install git

For Ubuntu, this PPA provides the latest stable upstream Git version

add-apt-repository ppa:git-core/ppa # apt update; apt install git

测试是否成功安装：

git –version

2. 环境配置

配置用户名和邮箱(最小配置)

local: 只对某个仓库有效
global: 对当前用户的所有仓库有效
system: 对系统所有登陆的用户有效

设置配置信息：

$ git config –global user.name “yourName”
$ git config –global user.email “yourEmail”

显示配置信息：

additional: use pwd to show current path

3. 创建仓库，配置local信息

两种场景:

  1. 已有项目纳入Git管理

  2. 新建项目直接用Git管理

设置local config文件

git config –local user.name “your name”
git config –local user.email “your email”

4. 工作区与缓存区

工作目录、暂存区、版本历史

工作目录 —-> git add files —-> 暂存区 —-> git commit —-> 版本历史

为什么要分add和commit两步走呢？
因为commit可以一次提交多个文件，所以可以先多次add文件，然后一次性commit
